# Фоторедактор — дизайн-спецификация

**Дата:** 2026-07-14
**Статус:** утверждён пользователем

## 1. Цель

Веб-сайт, куда пользователь загружает свои фотографии и редактирует их: удаляет/меняет
фон, улучшает (яркость/контраст/цвет/резкость), применяет трендовые фильтры-пресеты,
убирает мелкие лишние объекты кистью, кадрирует и поворачивает, затем скачивает результат.

**Ключевое ограничение (осознанный выбор):** всё считается **бесплатно и локально в
браузере**, без платного AI-API. Фотографии не покидают устройство пользователя.

## 2. Границы (что делаем / что нет)

**Делаем (хорошо работает бесплатно/в браузере):**
- Удаление и замена фона (прозрачный / цвет / своя картинка).
- Авто-улучшение + ручные ползунки (яркость, контраст, насыщенность, теплота, резкость).
- Трендовые фильтры-пресеты в один клик (кино, винтаж, тёплый, холодный, Ч/Б, матовый и т.п.).
- Лечащая кисть для **мелких** лишних объектов (провода, мусор, дефекты кожи).
- Кроп / поворот / ресайз под популярные форматы.
- История (undo/redo), сброс, сравнение «до/после».
- Экспорт в PNG (с прозрачностью) или JPG (с выбором качества).

**Не делаем (недостижимо бесплатно/локально в приемлемом качестве):**
- Генеративное удаление **крупных** объектов с реалистичным дорисовыванием.
- Полная «перерисовка/стилизация фото по референсу».
- Эти задачи требуют платного AI-мотора. Архитектуру закладываем так, чтобы такой
  мотор можно было добавить **отдельным модулем** позже, без переделки ядра.

## 3. Технический подход

- **Стек:** React + Vite + Tailwind CSS.
- **Архитектура:** полностью клиентское SPA. Бэкенда нет.
- **Хостинг:** статический билд, деплой на бесплатный статический хостинг
  (Netlify / Vercel / GitHub Pages). Одна команда сборки.
- **Приватность:** ни одно фото и ни один байт изображения не отправляется на сервер.
  Всё живёт в памяти вкладки.

## 4. Модули

Каждый инструмент — изолированный модуль с чёткой задачей и интерфейсом, работающий
поверх общего слоя-холста.

| Модуль | Задача | Технология (бесплатно, в браузере) |
|---|---|---|
| **background** | Удалить/заменить фон | `@imgly/background-removal` (нейросеть в браузере через WASM) → прозрачный PNG, затем подстановка цвета/картинки |
| **adjust** | Улучшение и ползунки | Canvas 2D: яркость/контраст/насыщенность/теплота/резкость; кнопка «Авто» |
| **filters** | Трендовые пресеты | Canvas 2D: набор готовых LUT/формул стилей в один клик |
| **heal** | Лечащая кисть | `OpenCV.js` `cv.inpaint` (Telea/NS): пользователь красит маску → алгоритм дорисовывает по соседним пикселям |
| **transform** | Кроп/поворот/ресайз | Canvas: обрезка под форматы (1:1, 4:5, 16:9, аватар), поворот, изменение размера |
| **history** | Undo/redo, сброс, «до/после» | Стек состояний изображения |
| **export** | Скачивание | Canvas → PNG (с альфой) или JPG (с ползунком качества) |

## 5. Поток данных

1. Пользователь загружает фото (drag&drop или выбор файла).
2. Изображение попадает в память как текущее состояние холста.
3. Любой инструмент читает **текущее** изображение и записывает **новый результат**
   в стек истории.
4. В любой момент доступны undo/redo, сравнение «до/после», сброс к оригиналу.
5. Экспорт берёт текущее состояние и отдаёт файл на скачивание.

Единый источник истины — текущее состояние холста + стек истории. Инструменты не
знают друг о друге, общаются только через это состояние.

## 6. Внешний вид (UX)

- Стиль: чистая современная «фотостудия», тёмная тема.
- Компоновка: крупная область предпросмотра по центру, панель инструментов сбоку,
  контекстные ползунки/пресеты для активного инструмента.
- Адаптивность: рабочая мобильная версия (сайт хостится онлайн, открывается с телефона).
- Обязательные элементы: кнопки undo/redo, «сбросить», переключатель «до/после»,
  кнопка экспорта.

## 7. Обработка ошибок и крайние случаи

- Загрузка не-изображения или битого файла → понятное сообщение, без падения.
- Очень большое фото → предупреждение/авто-даунскейл для производительности.
- Первая загрузка WASM-моделей (фон, OpenCV) → индикатор загрузки, пока веса
  скачиваются/инициализируются.
- Отсутствие результата inpaint (пустая маска) → подсказка «сначала закрасьте область».
- Экспорт без изменений → отдаём оригинал.

## 8. Тестирование

- Юнит-тесты на чистые функции обработки (adjust/filters — детерминированные
  преобразования пикселей на маленьких тестовых изображениях).
- Тест истории: undo/redo восстанавливает ожидаемые состояния.
- Тест экспорта: PNG сохраняет альфу, JPG применяет качество.
- Дымовой тест загрузки/инициализации тяжёлых модулей (фон, OpenCV) с моками.

## 9. Расширяемость на будущее

Модуль-контракт «инструмент получает изображение → возвращает изображение» позволяет
позже добавить платный AI-мотор (генеративное удаление крупных объектов, стилизация
по референсу) как ещё один модуль, не трогая ядро, историю и экспорт.
